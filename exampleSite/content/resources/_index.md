这里收录我日常真正会打开的工具和网站，按用途分组。不堆数量，只留高频且靠谱的。

## 监控与可观测

- [Prometheus](https://prometheus.io/) —— 云原生事实标准的指标采集与告警系统
- [Grafana](https://grafana.com/) —— 最流行的可视化看板，对接几乎所有数据源
- [OpenTelemetry](https://opentelemetry.io/) —— 统一的遥测数据标准，Metrics / Logs / Traces 三合一
- [Loki](https://grafana.com/oss/loki/) —— 像 Prometheus 一样思路的轻量日志聚合系统
- [Jaeger](https://www.jaegertracing.io/) —— 分布式链路追踪，定位跨服务延迟瓶颈

## CI/CD

- [GitLab](https://gitlab.com/) —— 一体化的代码托管 + CI/CD 平台
- [GitHub Actions](https://github.com/features/actions) —— 与 GitHub 深度集成的流水线，生态丰富
- [Argo CD](https://argo-cd.readthedocs.io/) —— Kubernetes 上的 GitOps 持续交付利器
- [Tekton](https://tekton.dev/) —— 云原生、声明式的流水线框架
- [Jenkins](https://www.jenkins.io/) —— 老牌 CI 工具，插件生态庞大

## Kubernetes 工具

- [k9s](https://k9scli.io/) —— 终端里的 K8s 管理 TUI，操作集群极顺手
- [Lens](https://k8slens.dev/) —— 图形化的 Kubernetes IDE
- [Helm](https://helm.sh/) —— Kubernetes 的包管理器，模板化部署
- [Kustomize](https://kustomize.io/) —— 无模板的声明式配置叠加工具
- [kubectx / kubens](https://github.com/ahmetb/kubectx) —— 快速切换集群上下文和命名空间
- [Karpenter](https://karpenter.sh/) —— 智能的 Kubernetes 节点弹性伸缩器

## 云平台

- [AWS](https://aws.amazon.com/) —— 全球最全的公有云，服务覆盖面最广
- [阿里云](https://www.aliyun.com/) —— 国内份额领先，本地化与备案友好
- [腾讯云](https://cloud.tencent.com/) —— 音视频、社交场景生态强
- [Cloudflare](https://www.cloudflare.com/) —— CDN、DNS、边缘计算与安全防护

## AI 工具

- [OpenAI](https://openai.com/) —— GPT 系列模型与 API
- [Anthropic Claude](https://www.anthropic.com/) —— Claude 系列模型，长上下文与代码能力强
- [Hugging Face](https://huggingface.co/) —— 开源模型与数据集的中心仓库
- [LangChain](https://www.langchain.com/) —— 构建 LLM 应用的编排框架

## 学习资源

- [CNCF Landscape](https://landscape.cncf.io/) —— 云原生全景图，技术选型必查
- [Roadmap.sh](https://roadmap.sh/) —— 各方向工程师的成长路线图
- [Awesome 系列](https://github.com/sindresorhus/awesome) —— 各领域精选资源的总入口
