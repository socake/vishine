跨云跑中间件，最怕的不是单个组件配错，而是 dev / staging / prod 之间的隔离不彻底，导致一个环境的变更串到另一个。

## 横向速查

- **消息队列**：阿里云 RocketMQ / AWS MSK / 自建 Kafka，注意 ACL 模型差异
- **缓存**：Redis / Valkey，改数据格式后必查脏数据
- **配置**：Nacos / Apollo，命名空间按环境严格隔离

## 隔离清单

1. 网络层：VPC / 安全组按环境分段
2. 凭据层：每环境独立 secret，禁止共享
3. 命名空间：MQ topic、配置 namespace 带环境前缀
